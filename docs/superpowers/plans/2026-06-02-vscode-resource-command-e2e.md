> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add VS Code E2E coverage for PR #17698's resource-command tree items after PR #17772's E2E harness lands, without merging or rebasing #17772 directly into #17698.

**Architecture:** Keep #17698's product behavior: `aspire describe --include-disabled-commands` feeds enabled and disabled API-visible resource commands into the tree, enabled command child items are executable from their context menu, disabled command child items render but are not executable, and CodeLens ignores disabled commands. After #17772 is merged to `main`, merge updated `main` into `e17698`, resolve the shared extension conflicts by preserving #17772's telemetry/E2E infrastructure and #17698's command-tree behavior, then add one E2E fixture command plus one UI-level tree assertion.

**Tech Stack:** TypeScript VS Code extension, ExTester-based VS Code E2E tests, Aspire AppHost C# fixture generated by `extension/scripts/run-e2e.js`, xUnit/MTP for CLI mapper tests.

- [ ] **Step 5: Resolve tree provider conflicts**

Keep #17698's command tree items:

```ts
class CommandsGroupItem extends vscode.TreeItem {
    constructor(public readonly resource: ResourceJson, public readonly resourceItem: ResourceItem, parentId: string) {
        super(commandsLabel, vscode.TreeItemCollapsibleState.Collapsed);
        this.id = `${parentId}:commands`;
        this.iconPath = new vscode.ThemeIcon('terminal');
        this.contextValue = 'commandsGroup';
    }
}

class ResourceCommandItem extends vscode.TreeItem {
    constructor(
        public readonly commandName: string,
        public readonly commandJson: ResourceCommandJson,
        public readonly resourceItem: ResourceItem,
        parentId: string
    ) {
        const label = commandJson.displayName ?? commandName;
        super(label, vscode.TreeItemCollapsibleState.None);
        this.id = `${parentId}:command:${commandName}`;
        this.tooltip = commandJson.description ?? undefined;

        const isEnabled = isEnabledCommand(commandJson);

        this.iconPath = getResourceCommandIcon(commandName, isEnabled);
        if (isEnabled) {
            this.contextValue = 'resourceCommand:enabled';
        } else {
            this.description = resourceCommandDisabledDescription;
            this.contextValue = 'resourceCommand:disabled';
        }
    }
}
```

Also keep #17772's AppHost-scoped command routing: resource command execution must use the clicked resource's owning AppHost, not the selected workspace AppHost.
